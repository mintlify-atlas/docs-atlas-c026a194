<Tabs>
  <Tab title="Docker">
    <Steps>
      <Step title="Run the Docker container">
        Make sure [Docker](https://docs.docker.com/get-docker/) is installed, then run:

        <CodeGroup>

        ```bash With OpenAI
        docker run -d --name dory \
          -p 3000:3000 \
          -e DS_SECRET_KEY="$(openssl rand -base64 32 | tr -d '\n')" \
          -e BETTER_AUTH_SECRET="$(openssl rand -hex 32)" \
          -e BETTER_AUTH_URL="http://localhost:3000" \
          -e DORY_AI_PROVIDER=openai \
          -e DORY_AI_MODEL=gpt-4o-mini \
          -e DORY_AI_API_KEY=your_api_key_here \
          -e DORY_AI_URL=https://api.openai.com/v1 \
          dorylab/dory:latest
        ```

        ```bash Without AI (SQL editor only)
        docker run -d --name dory \
          -p 3000:3000 \
          -e DS_SECRET_KEY="$(openssl rand -base64 32 | tr -d '\n')" \
          -e BETTER_AUTH_SECRET="$(openssl rand -hex 32)" \
          -e BETTER_AUTH_URL="http://localhost:3000" \
          dorylab/dory:latest
        ```

        </CodeGroup>

        <Note>
          `DS_SECRET_KEY` encrypts stored connection credentials (passwords, tokens). `BETTER_AUTH_SECRET` secures session tokens. Both are auto-generated in the commands above using `openssl`. For production, store these values persistently — rotating them invalidates existing sessions and stored connections.
        </Note>

        Once the container starts, open [http://localhost:3000](http://localhost:3000) in your browser.
      </Step>

      <Step title="Connect your first database">
        After logging in, click **New Connection** in the left sidebar.

        <Tabs>
          <Tab title="ClickHouse">
            Fill in the connection form:

            | Field    | Example value         |
            |----------|-----------------------|
            | Host     | `localhost`           |
            | Port     | `8123`                |
            | Database | `default`             |
            | Username | `default`             |
            | Password | *(leave blank if none)* |

            Click **Test Connection** to verify, then **Save**.

            <Tip>
              ClickHouse is the most deeply integrated database in Dory. You get real-time monitoring dashboards, query latency trends (P50/P95), and a full user and role management UI — in addition to the SQL editor and AI features.
            </Tip>
          </Tab>
          <Tab title="PostgreSQL">
            Fill in the connection form:

            | Field    | Example value              |
            |----------|----------------------------|
            | Host     | `localhost`                |
            | Port     | `5432`                     |
            | Database | `postgres`                 |
            | Username | `postgres`                 |
            | Password | your password              |

            Click **Test Connection** to verify, then **Save**.

            <Note>
              PostgreSQL support is currently in Beta. The SQL editor, AI Copilot, and schema-aware autocomplete are all supported.
            </Note>
          </Tab>
        </Tabs>
      </Step>

      <Step title="Open the SQL editor and run a query">
        Select your connection from the sidebar, then click **SQL Editor**.

        Type a query in the editor and press `Ctrl+Enter` (or `Cmd+Enter` on macOS) to run it:

        ```sql
        SELECT 1
        ```

        Results appear in the panel below the editor. You can open multiple tabs, each with its own query and result set.

        <Tip>
          Schema-aware autocomplete activates as you type. Start typing a table name or column name and Dory will suggest completions drawn from your live database schema.
        </Tip>
      </Step>

      <Step title="Enable AI features">
        If you started the container without `DORY_AI_API_KEY`, AI features (SQL Copilot and database chat) are disabled. To enable them, stop the container and re-run with the AI environment variables set:

        <CodeGroup>

        ```bash OpenAI
        docker run -d --name dory \
          -p 3000:3000 \
          -e DS_SECRET_KEY="your_existing_ds_secret_key" \
          -e BETTER_AUTH_SECRET="your_existing_auth_secret" \
          -e BETTER_AUTH_URL="http://localhost:3000" \
          -e DORY_AI_PROVIDER=openai \
          -e DORY_AI_MODEL=gpt-4o-mini \
          -e DORY_AI_API_KEY=sk-... \
          -e DORY_AI_URL=https://api.openai.com/v1 \
          dorylab/dory:latest
        ```

        ```bash Anthropic
        docker run -d --name dory \
          -p 3000:3000 \
          -e DS_SECRET_KEY="your_existing_ds_secret_key" \
          -e BETTER_AUTH_SECRET="your_existing_auth_secret" \
          -e BETTER_AUTH_URL="http://localhost:3000" \
          -e DORY_AI_PROVIDER=anthropic \
          -e DORY_AI_MODEL=claude-3-5-sonnet-20241022 \
          -e DORY_AI_API_KEY=sk-ant-... \
          dorylab/dory:latest
        ```

        ```bash Google Gemini
        docker run -d --name dory \
          -p 3000:3000 \
          -e DS_SECRET_KEY="your_existing_ds_secret_key" \
          -e BETTER_AUTH_SECRET="your_existing_auth_secret" \
          -e BETTER_AUTH_URL="http://localhost:3000" \
          -e DORY_AI_PROVIDER=google \
          -e DORY_AI_MODEL=gemini-1.5-flash \
          -e DORY_AI_API_KEY=your_google_ai_api_key \
          dorylab/dory:latest
        ```

        ```bash OpenAI-Compatible (Ollama, etc.)
        docker run -d --name dory \
          -p 3000:3000 \
          -e DS_SECRET_KEY="your_existing_ds_secret_key" \
          -e BETTER_AUTH_SECRET="your_existing_auth_secret" \
          -e BETTER_AUTH_URL="http://localhost:3000" \
          -e DORY_AI_PROVIDER=openai-compatible \
          -e DORY_AI_MODEL=llama3.2 \
          -e DORY_AI_API_KEY=ollama \
          -e DORY_AI_URL=http://host.docker.internal:11434/v1 \
          dorylab/dory:latest
        ```

        </CodeGroup>

        Once restarted with a valid `DORY_AI_API_KEY`, the **Copilot** panel and **Chat** button will become active in the SQL editor.

        <Note>
          Dory uses a pluggable AI provider architecture. You can switch between providers at any time by changing the `DORY_AI_PROVIDER`, `DORY_AI_MODEL`, `DORY_AI_API_KEY`, and `DORY_AI_URL` environment variables — no code changes required. See [AI providers](/ai/providers) for the full list.
        </Note>
      </Step>
    </Steps>
  </Tab>

  <Tab title="Desktop app">
    <Steps>
      <Step title="Download the desktop app">
        Download the latest release for your platform from the [Dory releases page](https://github.com/dorylab/dory/releases):

        - **macOS** — download the `.dmg` file, open it, and drag Dory to your Applications folder
        - **Windows** — download the `.exe` installer and run it

        <Note>
          The desktop app runs entirely on your machine using the `desktop` runtime (`DORY_RUNTIME=desktop`). No server or Docker installation is required.
        </Note>
      </Step>

      <Step title="Enable AI features">
        To use the SQL Copilot and database chat, open **Settings** and navigate to **AI Provider**.

        Enter your provider and API key:

        | Setting            | Example value                    |
        |--------------------|----------------------------------|
        | Provider           | `openai`                         |
        | Model              | `gpt-4o-mini`                    |
        | API key            | `sk-...`                         |
        | API URL            | `https://api.openai.com/v1`      |

        Save your settings. The **Copilot** panel and **Chat** button will activate in the SQL editor.

        <Note>
          See [AI providers](/ai/providers) for the full list of supported providers including Anthropic, Google Gemini, xAI Grok, Qwen, and any OpenAI-compatible endpoint.
        </Note>
      </Step>
