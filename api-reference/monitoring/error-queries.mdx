---
title: "Error queries"
description: "Retrieve queries that resulted in errors from ClickHouse."
---

<Note>
  Requires the `x-connection-id` header pointing to a ClickHouse connection with `queryInsights` capability. Data is sourced from ClickHouse's `system.query_log` table — Dory does not store query logs internally.
</Note>

Returns a paginated list of queries that resulted in errors within the selected time window. Use this to diagnose failed queries, identify recurring error patterns, and investigate database issues.

## Endpoint

```
POST /api/monitoring/error-queries
```

## Headers

<ParamField header="x-connection-id" type="string" required>
  ID of the active ClickHouse connection.
</ParamField>

## Request body

<ParamField body="pageIndex" type="number" default="0">
  Zero-based page index for pagination.
</ParamField>

<ParamField body="pageSize" type="number" default="10">
  Number of rows per page. Accepted values: `10`, `20`, `50`, `100`, `200`.
</ParamField>

<ParamField body="filters" type="object">
  Query insight filters to narrow the result set.
  <Expandable title="properties">
    <ResponseField name="timeRange" type="string" default="1h">
      Time window to query. One of `1h`, `6h`, `24h`, `7d`.
    </ResponseField>
    <ResponseField name="user" type="string" default="all">
      Filter by ClickHouse username. Use `all` to include all users.
    </ResponseField>
    <ResponseField name="database" type="string" default="all">
      Filter by database name. Use `all` to include all databases.
    </ResponseField>
    <ResponseField name="queryType" type="string" default="all">
      Filter by query type. One of `all`, `select`, `insert`, `ddl`, `other`.
    </ResponseField>
    <ResponseField name="minDurationMs" type="number" default="0">
      Minimum query duration in milliseconds. Use `0` to include all durations.
    </ResponseField>
    <ResponseField name="search" type="string" default="">
      Free-text search string applied to query text.
    </ResponseField>
  </Expandable>
</ParamField>

## Response

<ResponseField name="ok" type="boolean" required>
  `true` on success.
</ResponseField>

<ResponseField name="data" type="object" required>
  <Expandable title="properties" defaultOpen>
    <ResponseField name="rows" type="object[]" required>
      Page of error query log entries sourced from `system.query_log`. Common fields:
      <Expandable title="row fields">
        <ResponseField name="query_id" type="string">
          Unique query identifier assigned by ClickHouse.
        </ResponseField>
        <ResponseField name="query" type="string">
          The SQL text that resulted in an error.
        </ResponseField>
        <ResponseField name="user" type="string">
          ClickHouse user who executed the query.
        </ResponseField>
        <ResponseField name="query_start_time" type="string">
          Timestamp when the query started.
        </ResponseField>
        <ResponseField name="query_duration_ms" type="number">
          Duration in milliseconds before the error occurred.
        </ResponseField>
        <ResponseField name="exception" type="string">
          The error message returned by ClickHouse.
        </ResponseField>
        <ResponseField name="exception_code" type="number">
          Numeric ClickHouse error code.
        </ResponseField>
        <ResponseField name="stack_trace" type="string">
          Stack trace associated with the exception, if available.
        </ResponseField>
        <ResponseField name="type" type="string">
          Query log event type. Error entries have `type = 'ExceptionWhileProcessing'`.
        </ResponseField>
      </Expandable>
    </ResponseField>
    <ResponseField name="total" type="number" required>
      Total number of error queries matching the filter criteria (for pagination).
    </ResponseField>
  </Expandable>
</ResponseField>

## Error responses

| Status | Code | Description |
|---|---|---|
| `400` | `VALIDATION_ERROR` | Invalid filter parameters |
| `500` | `DATABASE_ERROR` | ClickHouse query failed or connection does not support query insights |

## Examples

<CodeGroup>

```bash Last hour of error queries
curl -X POST https://your-dory-instance/api/monitoring/error-queries \
  -H "Cookie: <session-cookie>" \
  -H "x-connection-id: conn_clickhouse_123" \
  -H "Content-Type: application/json" \
  -d '{
    "pageIndex": 0,
    "pageSize": 20,
    "filters": {
      "timeRange": "1h",
      "user": "all",
      "database": "all",
      "queryType": "all"
    }
  }'
```

```bash Filter by user and time range
curl -X POST https://your-dory-instance/api/monitoring/error-queries \
  -H "Cookie: <session-cookie>" \
  -H "x-connection-id: conn_clickhouse_123" \
  -H "Content-Type: application/json" \
  -d '{
    "pageIndex": 0,
    "pageSize": 50,
    "filters": {
      "timeRange": "24h",
      "user": "analyst",
      "database": "analytics"
    }
  }'
```

</CodeGroup>

```json 200
{
  "ok": true,
  "data": {
    "rows": [
      {
        "query_id": "8a2b3c4d-1234-5678-abcd-ef0123456789",
        "query": "SELECT * FROM non_existent_table WHERE id = 1",
        "user": "analyst",
        "query_start_time": "2024-06-01T10:15:30.000Z",
        "query_duration_ms": 12,
        "exception": "Table analytics.non_existent_table doesn't exist",
        "exception_code": 60,
        "stack_trace": "",
        "type": "ExceptionWhileProcessing"
      }
    ],
    "total": 1
  }
}
```

<Note>
  This endpoint is only available for ClickHouse connections. PostgreSQL and MySQL connections do not expose the `queryInsights` capability and will return an error.
</Note>
