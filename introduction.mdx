## What is Dory?

Dory is an AI-native data workspace built for engineers, analysts, and database platform teams. It brings together everything you need to work with modern databases — without switching between tools.

Unlike generic SQL clients, Dory's AI features are **grounded in your real database schema**. The SQL Copilot understands your actual tables and columns. The chatbot reads your connected database. Autocomplete suggests based on your live schema — not generic keywords.

## Deployment options

Dory runs in two modes:

- **Docker self-hosting** — One-command deployment, runs on your own infrastructure
- **Desktop app** — Native macOS and Windows app via Electron, no server needed

Both modes use the same `apps/web` application. The runtime is determined by the `DORY_RUNTIME` environment variable.

## AI provider support

Dory uses a pluggable AI provider architecture. Switch between providers by changing environment variables — no code changes required.

| Provider         | `DORY_AI_PROVIDER` value | Description                          |
|------------------|--------------------------|--------------------------------------|
| OpenAI           | `openai`                 | Default. Uses official OpenAI API.   |
| OpenAI-Compatible| `openai-compatible`      | Any OpenAI-compatible endpoint       |
| Anthropic        | `anthropic`              | Claude models via Anthropic API      |
| Google           | `google`                 | Gemini models via Google AI          |
| Qwen (Alibaba)   | `qwen`                   | Qwen models via DashScope            |
| xAI              | `xai`                    | Grok models via xAI API              |

<Note>
  AI features are optional. If `DORY_AI_API_KEY` is not set, the SQL Copilot and chatbot features will be disabled, but all other database tools remain fully functional.
</Note>
